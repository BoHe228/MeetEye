## 2. 横向展开，切片策略及代码说明

### 2.1 鱼眼横向展开

#### 2.1.1 核心原理

将圆形鱼眼图像通过极坐标变换展开为矩形全景图

```
鱼眼原图 (圆形)
    ↓
极坐标 → 直角坐标映射
    ↓
全景图 (矩形，360° × 垂直FOV)
```

#### 2.1.2 代码位置

主要实现：`panorama_processor.py` 和 `fisheye_panorama.py`

#### 2.1.3 映射矩阵计算

核心函数：`PanoramaProcessor.create_panorama_map()`

```python
def create_panorama_map(self):
    """
    创建横向展开的映射矩阵
    """
    # 对输出图像的每个像素:
    for y_out in range(output_height):
        for x_out in range(output_width):
            # 计算角度和半径比例
            angle = 2 * np.pi * x_out / output_width
            r_ratio = y_out / (output_height - 1)

            # 转换回原图坐标
            x_in = center_x + r_ratio * radius * cos(angle)
            y_in = center_y + r_ratio * radius * sin(angle)
```

#### 2.1.4 鱼眼区域检测

函数：`detect_fisheye_region_improved()`

策略：
1. 假设图像中心为圆心
2. 尝试霍夫圆检测
3. 尝试阈值+轮廓检测
4. 最终验证和修正圆心和半径

### 2.2 全景切片策略

#### 2.2.1 切片原理

将全景图水平切分为3个有重叠的切片，解决：
- 全景图过宽导致检测精度下降
- 边界目标被截断的问题

#### 2.2.2 切片策略详解

代码位置：`PanoramaSlicer.py:50`

**切片方案（3切片 + 10%重叠）：

```
全景图 (宽W, 高H)
    ↓
切片0: [W/3 - 10%, W*2/3 + 10%]
切片1: [W/3 - 10%, W*2/3 + 10%]
切片2: [W*2/3 - 10%, W + 10%]
        (注意: 超出边界的部分循环拼接)
```

**特殊处理：
- 切片0和切片2的边界环绕处理
- 首尾切片的重叠区域自动循环拼接

## 3. YOLO检测代码说明

### 3.1 YOLO检测器

代码位置：`yolo_pose.py`

### 3.2 检测流程

```python
def detect(self, image):
    # YOLO推理
    results = self.model(image, conf=self.conf_threshold)

    # 解析结果
    for result in results:
        boxes = result.boxes
        keypoints = result.keypoints
```

### 3.3 检测输出格式

每个检测包含：
- `bbox`: [x1, y1, x2, y2]
- `confidence`: 置信度
- `class_id`: 类别ID
- `keypoints`: 17个人体关键点
- `feature`: ReID特征向量（可选）

### 3.4 ⚠️ TensorRT 引擎的 task 推断（踩坑记录）

`.pt` 的 checkpoint 自带 `task` 元数据，而 `.engine` 文件不一定带；ultralytics
加载 `.engine` 猜不出 task 时会**默认按 `detect` 解析**，并打印警告：

```
WARNING ⚠️ Unable to automatically guess model task, assuming 'task=detect'.
```

**症状**：用 pose/face 引擎（如 `yolov8n-face.engine`）推理时报
`推理异常 [KeyError]: 13`（或其它数字），循环刷屏。

**根因**：pose/face 模型输出通道里含关键点坐标（face 5 点 = 20 通道，
pose 17 点 = 56 通道）。被当 `detect` 解析时，`4 个框通道` 之后的通道被误当作
"类别分数"做 argmax，得到一个越界的 `cls_id`（如 13），而
`result.names` 只有 `{0:'face'}` → `result.names[13]` → **KeyError: 13**
（`core/slicer.py` 与 `core/detector.py` 的 `result.names[int(cls_id)]` 处）。

**修复**（`core/detector.py` `YOLOPoseDetector.__init__`）：加载 `.engine` 时
按文件名显式指定 task，不依赖自动猜测：

```python
if model_path.lower().endswith('.engine'):
    _task = 'pose' if ('pose' in name or 'face' in name) else 'detect'
    self.model = YOLO(model_path, task=_task)   # 名含 pose/face → pose
else:
    self.model = YOLO(model_path)               # .pt/.onnx 自带 task
```

- `yolov8n-face.engine` / `*-pose.engine` → `pose` ✓
- `yolo26n.engine`（纯检测 / 补漏）→ `detect` ✓

**注意**：该推断依赖文件名含 `pose` / `face` 关键字。导出引擎时务必保持命名规范
（见 `yolo_model/export_trt.py`），否则 pose/face 引擎若改了名字会被判成 detect 再次踩坑。

#### 4.2.3 ReID特征相似度

使用余弦相似度：

```python
def cosine_similarity(vec1, vec2):
    return dot(vec1, vec2) / (norm(vec1) * norm(vec2))
```

只有相似度 > 0.7 认为是同一目标

def check_new_track(self, bbox, feature, ...):
    """
    新目标在边界出现时，寻找匹配的消失目标
    """
    appear_side = get_boundary_side(bbox)
