# 角度计算流程说明

## 概述

系统通过 YOLO-Pose 检测到人体关键点后，取鼻尖在全景图中的像素坐标，分别计算出该人相对摄像头的**水平角（azimuth）**和**俯仰角（elevation）**，最终序列化进 JSON 发送给客户端。

---

## 完整流程图

```
YOLO 关键点 (全景图像素坐标)
         │
         ▼
   取鼻尖像素坐标 (nose_x, nose_y)
         │
    ┌────┴──────────────────────────┐
    │                               │
    ▼                               ▼
 水平角 azimuth               俯仰角 elevation
    │                               │
    │  线性映射                      │  两步转换
    │  x → [0°, 360°)              │
    │                        ┌──────┴──────┐
    │                        ▼             │
    │                  全景坐标            │
    │                  → 鱼眼坐标          │
    │                  (查映射表)          │
    │                        │             │
    │                        ▼             │
    │                  鱼眼像素半径 r      │
    │                  = dist(px, py, 圆心)│
    │                        │             │
    │                        ▼             │
    │                  多项式标定公式       │
    │                  elevation = poly(r) │
    │                        └──────┬──────┘
    │                               │
    └──────────┬────────────────────┘
               ▼
     { azimuth_deg, elevation_deg }
      序列化进 JSON → 发送给 angle_visualizer.py
```

---

## 各步骤代码说明

### Step 1：取鼻尖坐标

**文件**：`core/angle_calculator.py` → `calculate_angles_from_keypoints()`

```python
# COCO 关键点格式：kpt[0] = 鼻子
nose_x, nose_y = float(nose_kp[0]), float(nose_kp[1])
# 坐标是在裁剪后的全景图中（已减去 crop_top_offset）
```

角度计算的输入始终是**鼻尖**位置，不是人体中心或其他关键点。
坐标参考系是裁剪后的全景图（若设置了顶部裁剪，坐标已相应偏移）。

---

### Step 2：水平角（azimuth）

**文件**：`core/angle_calculator.py` → `_pixel_to_angle()`

```python
# 全景图 x 方向是 360° 均匀展开的
azimuth_deg = 360.0 * (x / self.panorama_width)  # 0° ~ 360°
```

**原理**：鱼眼展开为全景图时，水平方向是对方位角的线性映射，全景图最左列 = 0°，最右列 ≈ 360°，因此直接按比例换算即可，不需要标定。

---

### Step 3：全景坐标 → 鱼眼坐标

**文件**：`core/angle_calculator.py` → `panorama_to_fisheye()`

```python
# 加回顶部裁剪偏移，还原为完整全景中的行索引
yi = int(y_panorama + self.crop_top_offset)
xi = int(x_panorama)

# 直接查预计算映射表（展开时生成，避免重复三角运算）
x_fisheye = panorama_map_x[yi, xi]
y_fisheye = panorama_map_y[yi, xi]
```

映射表（`map_x` / `map_y`）在系统启动时由 `FisheyePanoramaGPU` 生成，记录了全景图每个像素对应的原始鱼眼图坐标，查表替代逐帧的三角函数计算。

---

### Step 4：俯仰角（elevation）

**文件**：`core/angle_calculator.py` → `FisheyeConverter`

```python
# Step 4a：计算鱼眼图中到圆心的像素半径
r = sqrt((x_fisheye - cx)² + (y_fisheye - cy)²)

# Step 4b：用标定多项式将半径映射到仰角
# 系数由实际场景标定数据拟合（4次或5次多项式，R²≈0.9995）
elevation_deg = polyval(coeffs, r)
```

**原理**：鱼眼镜头的成像遵循等距投影（equidistant projection），像素到圆心的半径与入射光线的天顶角呈近似多项式关系，通过真实场景标定拟合出系数，半径越大对应仰角越小（图像边缘对应地面方向）。

默认内置系数（5次多项式，R²≈0.9996）：

```python
coeffs = [
    -1.1658e-11,
     1.1878e-08,
    -4.2396e-06,
     5.6684e-04,
    -1.9621e-01,
    90.564
]
```

也可通过 `--calib-yaml` 参数加载外部标定文件替换内置系数。

---

### Step 5：打包发送

**文件**：`main_GPU_webui.py` → `_build_inference_json()`

```python
targets[str(tid)] = {
    'azimuth':        round(angle['azimuth_deg'],   3),
    'elevation':      round(angle['elevation_deg'], 3),
    'eye_pixel_dist': round(eye_d, 2),         # 原始双眼像素距离（监控用）
    'distance':       round(distance, 3),       # 姿态修正后的物理距离（m）
    'features':       feat_list,                # ReID 外观特征向量
}
```

## 关键约束与误差来源

| 项目 | 说明 |
|------|------|
| 水平角精度 | ≈ 360 / panorama_width，3840px 下约 0.09°/px |
| 俯仰角精度 | 取决于标定质量，R²≈0.9995，边缘区域误差略大 |
| 角度基准点 | 鼻尖，不是人体站立位置，俯仰角会略偏高 |
| 水平角零点 | 全景图最左列 = 0°，需与相机实际安装朝向对齐 |
| 裁剪偏移 | 若启用顶部裁剪（`--crop-divisor`），俯仰角计算时会自动加回偏移量还原正确坐标 |

---

## 涉及文件

| 文件 | 作用 |
|------|------|
| `core/angle_calculator.py` | 角度计算核心逻辑（水平角、俯仰角、鱼眼坐标转换） |
| `webui/processor.py` | 调用角度计算，将结果附加到 `angle_info` |
| `main_GPU_webui.py` | 将角度与距离打包序列化为 JSON |
| `angle_visualizer.py` | 客户端接收 JSON，在 3D 半球体和雷达图上可视化 |
